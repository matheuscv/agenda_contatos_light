Arquitetura da Agenda de Contatos

Como um frontend Next.js se integra a um servidor MCP externo de forma segura.

O problema central

O browser não consegue falar diretamente com o servidor MCP. Dois motivos:

  • CORS — o servidor MCP externo não permite chamadas de origens aleatórias.
  • Segurança — a MCP_API_KEY não pode ficar exposta no bundle JavaScript que o browser baixa.

A solução é colocar um intermediário servidor no meio.

O fluxo completo

Browser (cliente)
  • React / Next.js — componentes client
  • page.tsx · ContactsTable · ContactDialog
  • Faz fetch("/api/contacts") — URL relativa, mesma origem
HTTP / JSON (mesma origem — sem CORS)
API Routes Next.js — roda no servidor Vercel
  • /api/contacts → GET listar · POST criar
  • /api/contacts/[id] → PUT editar · DELETE excluir
  • Aqui vivem MCP_SERVER_URL e MCP_API_KEY (variáveis de ambiente secretas)
  • Chama callTool() de lib/mcp.ts
Protocolo MCP — SSE / HTTP · Authorization: Bearer <token>
lib/mcp.ts — cliente MCP (SDK)
  • Instancia Client do @modelcontextprotocol/sdk
  • Conecta via StreamableHTTPClientTransport (ou SSE como fallback)
  • Invoca a tool, fecha a conexão, retorna o resultado como JSON
Invocação de tool autenticada
Servidor MCP externo — contatos-mcp.onrender.com
  • Expõe tools: listar_contatos · criar_contato · atualizar_contato · excluir_contato
  • Valida o Bearer token em cada chamada
  • Banco de dados real fica aqui — a aplicação não tem BD próprio

Cada camada em detalhe

1

Frontend (browser)

São componentes React normais. Não sabem nada sobre MCP — só fazem fetch para URLs da própria aplicação:

// Lista contatos
const res = await fetch("/api/contacts");

// Cria contato
const res = await fetch("/api/contacts", { method: "POST", body: ... });

// Exclui contato
const res = await fetch("/api/contacts/42", { method: "DELETE" });

O browser acha que está falando com a própria aplicação. O MCP é completamente invisível para ele.

2

API Routes (proxy servidor)

São funções Node.js que rodam no servidor da Vercel, nunca no browser. Recebem a chamada do frontend e traduzem para o protocolo MCP:

// app/api/contacts/route.ts
export async function GET() {
  const data = await callTool("listar_contatos", {});
  return NextResponse.json(data);
}

export async function POST(request: Request) {
  const { nome, email, empresa } = await request.json();
  const data = await callTool("criar_contato", { nome, email });
  if (empresa && data?.id)
    await callTool("atualizar_contato", { id: data.id, empresa });
  return NextResponse.json(data, { status: 201 });
}

É aqui que a MCP_API_KEY é usada — ela existe apenas neste ambiente de servidor, nunca chega ao browser.

3

lib/mcp.ts (cliente MCP)

É o "tradutor de protocolo". Recebe um nome de tool + argumentos e cuida de toda a comunicação com o servidor MCP:

export async function callTool(name: string, args = {}) {
  const client = new Client({ name: "agenda-contatos", version: "1.0.0" });
  await client.connect(transport);           // handshake MCP
  const result = await client.callTool({ name, arguments: args });
  await client.close();                        // fecha — serverless friendly
  return JSON.parse(result.content[0].text);  // retorna dado limpo
}

O SDK @modelcontextprotocol/sdk cuida do handshake, do formato das mensagens e do transporte — nós só chamamos callTool().

4

Servidor MCP externo

É um servidor independente (hospedado no Render.com) que expõe ferramentas no lugar de endpoints REST tradicionais. Ele tem o banco de dados real e valida o token de autenticação em cada chamada.

As tools disponíveis foram descobertas em desenvolvimento via client.listTools() e mapeadas nas API Routes:

Rota Next.jsMétodoTool MCP
/api/contactsGETlistar_contatos
/api/contactsPOSTcriar_contato + atualizar_contato
/api/contacts/[id]PUTatualizar_contato
/api/contacts/[id]DELETEexcluir_contato

Por que esse padrão funciona bem

ProblemaSolução adotada
MCP_API_KEY não pode ir ao browser Fica só nas API Routes (variável de ambiente de servidor)
CORS bloqueia chamadas diretas ao MCP Browser fala só com /api/* — mesma origem, sem CORS
Protocolo MCP é complexo (SSE, handshake) SDK encapsula tudo em uma única função callTool()
Vercel é serverless — sem conexão persistente callTool() abre e fecha a conexão a cada chamada
MCP rejeitava empresa na criação Criamos o contato sem o campo e fazemos um PUT imediato

Resumo em uma frase

O browser fala HTTP comum com as API Routes do Next.js; as API Routes falam protocolo MCP com o servidor externo usando uma chave secreta — o frontend nunca vê nem a chave nem o protocolo.
